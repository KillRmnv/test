
## 1. Файловая система сервера (локальное хранилище)

### 1.1 Обычные файлы на диске

**Как:**  
Файлы сохраняются в директориях (`/uploads`, `/data`, `/media`).

**Используется для:**

- изображений
    
- видео
    
- аудио
    
- больших текстов (PDF, TXT)
    

**Плюсы:**

- просто
    
- быстро
    
- нативная работа ОС
    
- минимум зависимостей
    

**Минусы:**

- плохо масштабируется
    
- сложно в кластере
    
- резервное копирование — на вашей ответственности
    

**Типично:**  
`Nginx + PHP / Node / Java`  
`/uploads/user_id/file.ext`

---

### 1.2 Виртуальные файловые системы

**Примеры:**

- FUSE
    
- overlayfs
    

Используется редко напрямую в вебе, чаще внутри инфраструктуры.

---

## 2. База данных (BLOB / CLOB)

### 2.1 Хранение файлов как BLOB

**Как:**  
Файл хранится в поле базы данных.

**Поддерживается:**

- PostgreSQL (`BYTEA`, Large Objects)
    
- MySQL (`BLOB`, `LONGBLOB`)
    
- Oracle
    
- SQLite
    

**Плюсы:**

- атомарность
    
- транзакции
    
- единое хранилище
    

**Минусы:**

- нагрузка на БД
    
- плохая масштабируемость
    
- тяжело стримить видео
    

**Используется для:**

- небольших файлов
    
- документов
    
- аватаров
    

---

### 2.2 Хранение больших текстов (CLOB)

Для:

- статей
    
- логов
    
- расшифровок аудио
    
- JSON / XML
    

---

## 3. Объектные хранилища (Object Storage) ⭐ **стандарт де-факто**

### 3.1 Облачные

**Примеры:**

- Amazon S3
    
- Google Cloud Storage
    
- Azure Blob Storage
    
- Yandex Object Storage
    

**Как:**

- файл = объект
    
- доступ по URL
    
- метаданные отдельно
    

**Плюсы:**

- масштабируемость
    
- высокая доступность
    
- дешёвое хранение
    
- CDN-дружественно
    

**Минусы:**

- задержки
    
- зависимость от облака
    

**Идеально для:**

- видео
    
- аудио
    
- изображения
    
- большие архивы
    

---

### 3.2 Self-hosted объектные хранилища

**Примеры:**

- MinIO
    
- Ceph
    
- OpenStack Swift
    

Используются в enterprise и on-premise.

---

## 4. Distributed File Systems (DFS)

### 4.1 Сетевые файловые системы

**Примеры:**

- NFS
    
- SMB
    

**Как:**

- общий каталог для всех серверов
    

**Плюсы:**

- просто для кластера
    

**Минусы:**

- узкое место
    
- проблемы с отказоустойчивостью
    

---

### 4.2 Распределённые FS

**Примеры:**

- CephFS
    
- GlusterFS
    
- HDFS
    

**Используется для:**

- больших медиа-платформ
    
- аналитики
    
- стриминга
    

---

## 5. Специализированные хранилища по типу данных

### 5.1 Видео-хранилища и стриминг

**Подходы:**

- chunked upload
    
- HLS / DASH
    
- хранение сегментами
    

**Типично:**

- видео → object storage
    
- метаданные → БД
    

---

### 5.2 Аудио

- object storage
    
- иногда сжатые бинарные форматы
    
- часто с CDN
    

---

### 5.3 Изображения

- object storage
    
- кеширование
    
- версии (thumbnail, original)
    

---

## 6. Content-Addressable Storage (CAS)

**Как:**

- файл хранится по хешу (SHA-256)
    

**Примеры:**

- IPFS
    
- Git-подобные хранилища
    

**Плюсы:**

- дедупликация
    
- неизменяемость
    

**Минусы:**

- сложность
    
- не всегда подходит для веба
    

---

## 7. Кэш-хранилища (НЕ основное)

### 7.1 Redis / Memcached

Используется:

- для временного хранения
    
- превью
    
- буферов загрузки
    

❌ **не для постоянных файлов**

---

## 8. Гибридные схемы ⭐ **на практике чаще всего**

### 8.1 Самая популярная архитектура

```
Файл → Object Storage (S3)
Метаданные → БД
Кэш → CDN / Redis
```

### 8.2 Альтернатива для малого проекта

```
Файл → локальный диск
Метаданные → БД
```

**Где шифруют:**

- на уровне приложения
