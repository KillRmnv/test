**SPARQL** (произносится как «спаркл», расшифровывается как *SPARQL Protocol and RDF Query Language*) — это язык запросов для баз данных, хранящих информацию в формате **RDF** (Resource Description Framework).

Простыми словами: **если SQL — это язык для работы с таблицами (реляционными базами), то SPARQL — это язык для работы с графами (семантическими базами данных).**

Вот подробный разбор того, что нужно знать о SPARQL.

---

### 1. Контекст: Что такое RDF?
Чтобы понять SPARQL, нужно понимать, как хранятся данные. В обычных базах данных (SQL) данные лежат в таблицах со строками и столбцами. В мире SPARQL данные хранятся в виде **триплетов (троек)**:

> **Субъект — Предикат — Объект**
> *(Subject — Predicate — Object)*

**Пример:**
*   **Субъект:** `Лев_Толстой`
*   **Предикат:** `написал`
*   **Объект:** `Война_и_Мир`

Визуально это выглядит как граф: узлы (субъекты и объекты) соединены ребрами (предикатами). SPARQL позволяет искать пути и закономерности в этом графе.

---

### 2. Основы синтаксиса SPARQL
Запрос на SPARQL состоит из шаблонов, которые сопоставляются с данными в базе.

#### Переменные
В SPARQL переменные начинаются со знака вопроса `?` или доллара `$`.
*   `?author` — переменная для автора.
*   `?book` — переменная для книги.

#### Базовая структура запроса
Самый частый тип запроса — `SELECT`.

```sparql
PREFIX ex: <http://example.org/>

SELECT ?book ?title
WHERE {
  ?book ex:author ex:Tolstoy .
  ?book ex:title ?title .
}
```

**Разбор:**
1.  **PREFIX:** Сокращение для длинных URI (адресов ресурсов), чтобы не писать их каждый раз полностью.
2.  **SELECT:** Какие переменные мы хотим увидеть в результате.
3.  **WHERE:** Здесь описывается **паттерн** (шаблон).
    *   Строка `?book ex:author ex:Tolstoy .` означает: «Найди мне такую книгу (`?book`), у которой автор — Толстой».
    *   Точка `.` в конце строки означает конец триплета.

---

### 3. Типы запросов в SPARQL
В отличие от SQL, где в основном используют `SELECT`, в SPARQL есть 4 основных формы запросов:

1.  **SELECT:** Возвращает таблицу результатов (переменные и значения). Аналог `SELECT` в SQL.
2.  **CONSTRUCT:** Возвращает новый RDF-граф на основе найденных данных. Полезно для трансформации данных.
3.  **ASK:** Возвращает простой булевый ответ (`true`/`false`). Например: «Существует ли книга с таким названием?».
4.  **DESCRIBE:** Возвращает RDF-описание конкретного ресурса (все известные триплеты о нем).

---

### 4. Продвинутые возможности
SPARQL 1.1 (текущий стандарт) поддерживает мощные конструкции:

*   **FILTER:** Фильтрация результатов.
    ```sparql
    FILTER (?year > 1900)
    ```
*   **OPTIONAL:** Аналог `LEFT JOIN` в SQL. Если данных нет, запрос не падает, а переменная остается пустой.
    ```sparql
    OPTIONAL { ?book ex:translator ?translator }
    ```
*   **UNION:** Объединение результатов разных паттернов (аналог `OR`).
*   **Агрегация:** `COUNT`, `SUM`, `AVG`, `GROUP BY`.
    ```sparql
    SELECT (COUNT(?book) AS ?count)
    WHERE { ?book ex:author ex:Tolstoy }
    ```
*   **Обновление данных:** SPARQL 1.1 позволяет делать `INSERT`, `DELETE`, `LOAD` (хотя не все базы данных поддерживают запись через SPARQL).

### 5. Где это используется? (Реальные примеры)
Самый известный публичный пример — **Wikidata** (база знаний, стоящая за Википедией). У них есть сервис **Wikidata Query Service**, где можно писать запросы на SPARQL.

**Пример запроса к Wikidata:**
*Найти всех женщин-писательниц из России, родившихся в 19 веке.*

```sparql
SELECT ?writer ?writerLabel ?birthDate
WHERE {
  ?writer wdt:P31 wd:Q5;          # Экземпляр: Человек
          wdt:P21 wd:Q6581097;    # Пол: Женский
          wdt:P27 wd:Q159;        # Гражданство: Россия
          wdt:P106 wd:Q36180;     # Род занятий: Писатель
          wdt:P569 ?birthDate.    # Дата рождения
  
  FILTER((?birthDate >= "1800-01-01T00:00:00Z") && 
         (?birthDate <= "1899-12-31T23:59:59Z"))
  
  SERVICE wikibase:label { 
    bd:serviceParam wikibase:language "[AUTO_LANGUAGE],ru". 
  }
}
```
*Этот запрос вернет список имен и дат рождения.*

### 6. Инструменты и Базы Данных
Если вы хотите попробовать SPARQL, вам понадобятся:

**Публичные эндпоинты (где можно писать запросы онлайн):**
1.  **Wikidata Query Service** (query.wikidata.org) — огромная база знаний.
2.  **DBpedia** (dbpedia.org/sparql) — структурированные данные из Википедии.
3.  **Europeana** — данные о культурном наследии Европы.

**Серверы (Triplestores) для развертывания у себя:**
1.  **Apache Jena Fuseki** (популярный, Java).
2.  **GraphDB** (мощный, есть бесплатная версия).
3.  **Virtuoso** (высокая производительность).
4.  **Blazegraph** (быстрый, используется в Wikidata).

### 7. Плюсы и Минусы

**Плюсы:**
*   **Гибкость схемы:** Не нужно заранее проектировать жесткую структуру таблиц. Данные могут быть разнородными.
*   **Связность:** Идеально для данных, где важны связи между объектами (соцсети, знания, биология).
*   **Стандарт W3C:** Единый стандарт для Semantic Web (Семантического веба).
*   **Федеративные запросы:** Можно делать запрос сразу к нескольким разным базам данных одновременно.

**Минусы:**
*   **Порог входа:** Синтаксис сложнее для новичка, чем SQL. Нужно понимать концепцию URI и графов.
*   **Производительность:** На очень больших графах сложные запросы могут работать медленно (хотя современные движки очень быстрые).
*   **Распространенность:** Меньше разработчиков знают SPARQL по сравнению с SQL.

### Итог
**SPARQL** — это ключевой инструмент для работы со связанными данными (Linked Data) и знаниями. Если ваша задача — искать сложные связи между сущностями, интегрировать данные из разных источников без единой схемы или работать с онтологиями, SPARQL — это лучший выбор.
