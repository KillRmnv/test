### 1. Направленность (Directionality)

В Neo4j **все отношения являются направленными** (Directed). Это основное и обязательное свойство отношений.

|**Тип**|**Описание**|**Синтаксис Cypher**|
|---|---|---|
|**Направленное**|Отношение имеет четкое начало и конец. Это фундаментальное требование для всех связей в Neo4j.|`(a)-[R]->(b)` или `(a)<-[R]-(b)`|

**Ненаправленность в Cypher (для запросов):**

Хотя все отношения в базе данных хранятся как направленные, при написании запросов на Cypher вы можете **игнорировать направленность** (искать связь вне зависимости от того, куда она идет).

|**Тип**|**Описание**|**Синтаксис Cypher**|
|---|---|---|
|**Ненаправленное (в запросе)**|Используется для поиска шаблона, не зависящего от направления связи.|`(a)-[R]-(b)`|

**Пример из вашего запроса:**

Cypher

```
MATCH (people:Person)-[relatedTo]-(:Movie {title: "Cloud Atlas"})
```

В этом запросе, `-[]-` без стрелки, означает "найди всех `Person`, которые связаны с фильмом `Cloud Atlas` либо через `->`, либо через `<-`".

### 2. Свойства отношений (Weighted / Properties)

В общей теории графов "весовые" (weighted) графы — это те, где отношения имеют числовой вес (например, стоимость, расстояние). В Neo4j это реализуется через **свойства отношений**.

Neo4j не имеет специального типа "весовое отношение", но позволяет хранить любые данные (включая числа) непосредственно на связи.

|**Тип**|**Описание**|**Пример из вашего запроса**|
|---|---|---|
|**Свойственное / Весовое**|Отношение может содержать произвольные пары ключ-значение. Это позволяет хранить _вес_ (`weight`), _дату_ (`since`), _роль_ (`roles`) или _рейтинг_ (`rating`) на самой связи.|`-[:KNOWS {since: 2001}]->`|

**Примеры из вашего запроса:**

- Отношение `ACTED_IN` имеет свойство `roles` (роли):
    
    Cypher
    
    ```
    RETURN people.name, Type(relatedTo), relatedTo.roles
    ```
    
- Ваш личный пример отношения `KNOWS` имеет свойство `since: 2001`, которое можно считать весом (временным весом).
