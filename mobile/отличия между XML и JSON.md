**XML (eXtensible Markup Language)** и **JSON (JavaScript Object Notation)** — это два самых популярных формата для структурирования, хранения и передачи данных между системами.

Хотя они решают одну и ту же задачу, подходы к структурированию информации у них принципиально разные. JSON проектировался как минималистичный формат для веба, в то время как XML создавался как универсальный, строгий язык разметки документов.

### 1. Как выглядят одни и те же данные

Представим, что нам нужно передать информацию о студенте Кирилле, который учится в БГУИР на специальности ИИ:

#### Данные в формате JSON:

JSON

```
{
  "student": {
    "name": "Кирилл",
    "university": "БГУИР",
    "specialty": "Искусственный интеллект",
    "isStudent": true,
    "courses": [1, 2, 3]
  }
}
```

#### Данные в формате XML:

XML

```
<?xml version="1.0" encoding="UTF-8"?>
<student>
    <name>Кирилл</name>
    <university>БГУИР</university>
    <specialty>Искусственный интеллект</specialty>
    <isStudent>true</isStudent>
    <courses>
        <course>1</course>
        <course>2</course>
        <course>3</course>
    </courses>
</student>
```

### 2. Ключевые различия

|**Критерий**|**JSON**|**XML**|
|---|---|---|
|**Смысловая концепция**|Формат описания **объектов** и структур данных.|Язык **разметки** документов (похож на HTML).|
|**Читаемость (Human-readability)**|Очень легкий, чистый синтаксис без лишнего визуального шума.|Более тяжелый синтаксис из-за обязательных закрывающихся тегов.|
|**Размер данных (Overhead)**|**Компактный.** Передается меньше избыточных символов, что экономит трафик.|**Избыточный.** Закрывающие теги существенно увеличивают размер файла.|
|**Поддержка типов данных**|**Есть.** Различает строки, числа, boolean, массивы, объекты и null.|**Нет.** Всё является строковым текстом (значения `true` или `1` парсер изначально видит как обычные строки).|
|**Валидация схемы**|Опционально (через JSON Schema, используется реже).|**Строгая из коробки** (через DTD или XSD схемы). Позволяет жестко валидировать структуру документа.|
|**Парсинг (Чтение кодом)**|Парсится мгновенно встроенной в браузеры и языки функцией `JSON.parse()`.|Требует более сложных и ресурсоемких DOM/SAX-парсеров.|
|**Дополнительные фичи**|Нет. Предназначен строго для транспортировки данных.|Поддерживает пространства имен (namespaces), комментарии, CDATA и XPath-запросы для навигации.|

### 3. Сильные и слабые стороны

#### Плюсы JSON:

1. **Скорость и легкость:** Браузеры обрабатывают JSON в разы быстрее, чем XML. Это критично для SPA-приложений (React, Angular, Vue).
    
2. **Идеальное соответствие ООП:** JSON-структура один в один мапится на объекты в современных языках программирования (включая Java-классы через библиотеки вроде Jackson или Gson).
    

#### Плюсы XML:

1. **Строгие контракты (XSD):** В корпоративных системах (например, в SOAP-протоколах) очень важно, чтобы структура документа проверялась до её обработки. XML-схема гарантирует, что если в документе пропущено обязательное поле, парсер выдаст ошибку еще на этапе приема файла.
    
2. **Сложная разметка:** Если тебе нужно передать не просто структуру базы данных, а отформатированную статью с картинками, стилями и ссылками, XML справится с этим намного лучше.
    

### 💡 Как ответить на собеседовании:

> _«Главное отличие — в философии и избыточности. JSON ориентирован на данные: он компактный, быстрый, поддерживает нативные типы данных (числа, массивы, логический тип) и является стандартом в современных REST API._
> 
> _XML — это язык разметки документов. Он избыточен из-за закрывающихся тегов, но предлагает строгую валидацию структуры с помощью XML-схем (XSD) и поддержку комментариев, поэтому до сих пор доминирует в SOAP-сервисах, финтехе и банковских интеграциях, где надежность контракта важнее скорости передачи»_.
