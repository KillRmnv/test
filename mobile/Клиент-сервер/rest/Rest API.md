## 🔷 Что такое REST API

**REST (Representational State Transfer)** — архитектурный стиль построения веб-сервисов, при котором:

> ресурсы идентифицируются URL, а действия над ними выполняются стандартными HTTP-методами.

REST — **не протокол**, а **набор принципов** поверх HTTP.

---

## 🔷 Ключевые принципы REST

1. **Client–Server**  
    Клиент и сервер независимы.
    
2. **Stateless**  
    Сервер **не хранит состояние клиента**. Каждый запрос самодостаточный.
    
3. **Cacheable**  
    Ответы могут кэшироваться.
    
4. **Uniform Interface (главное!)**  
    Единый способ работы с ресурсами через:
    
    - URL
        
    - HTTP методы
        
    - коды ответов
        
    - представления (JSON/XML)
        
5. **Layered System**  
    Возможны прокси, балансировщики, gateway — клиент об этом не знает.
    
6. **Code on Demand (необязательно)**  
    Сервер может прислать исполняемый код (редко используется).
    

---

## 🔷 Что такое «ресурс»

Ресурс — это **сущность системы**, например:

```
/users
/orders
/products/15
```

**Не действия!** ❌

```
/createUser
/getAllOrders
/deleteProduct
```

---

## 🔷 HTTP методы в REST

|Метод|Назначение|Идемпотентность|
|---|---|---|
|GET|Получить ресурс|✅|
|POST|Создать ресурс|❌|
|PUT|Полностью обновить|✅|
|PATCH|Частично обновить|❌|
|DELETE|Удалить|✅|

---

## 🔷 Пример правильного REST API

|Действие|Метод|URL|
|---|---|---|
|Получить всех пользователей|GET|`/users`|
|Получить одного|GET|`/users/10`|
|Создать|POST|`/users`|
|Обновить|PUT|`/users/10`|
|Частично обновить|PATCH|`/users/10`|
|Удалить|DELETE|`/users/10`|

## 🔷 Формат обмена

Обычно:

## 🔷 Коды ответов (очень любят спрашивать)

|Код|Когда|
|---|---|
|200 OK|Успешный GET/PUT/PATCH|
|201 Created|Успешный POST|
|204 No Content|Удаление|
|400 Bad Request|Ошибка запроса|
|401 Unauthorized|Нет авторизации|
|403 Forbidden|Нет прав|
|404 Not Found|Ресурс не найден|
|500 Internal Server Error|Ошибка сервера|

---

## 🔷 Как понять, что API «по-настоящему RESTful»

Если:

- в URL только **существительные**
    
- используются **HTTP методы по назначению**
    
- нет хранения состояния
    
- используются **коды HTTP**
    
- возвращается представление ресурса (JSON)
    

👉 значит это REST.

Если видишь:

```
/api/createUser
/api/deleteUser?id=5
```

— это уже **не REST**, а RPC поверх HTTP.

## 🔷 Одна фраза для экзамена
