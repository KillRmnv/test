
## 🔷 Что такое Protobuf

**Protocol Buffers (protobuf)** — это **бинарный формат сериализации данных** и язык описания структур, разработанный Google.

> Позволяет **компактно и быстро передавать данные между сервисами**, с строгой типизацией и поддержкой версий.

---

## 🔷 Главная идея

Вместо того чтобы передавать данные как JSON или XML:

```json
{
  "id": 10,
  "name": "John",
  "email": "john@example.com"
}
```

Protobuf сериализует в **бинарный компактный формат**, который:

- меньше по размеру
    
- быстрее обрабатывается
    
- строго типизирован
    

---

## 🔷 Как это выглядит в `.proto` файле

```proto
syntax = "proto3";

message User {
  int32 id = 1;
  string name = 2;
  string email = 3;
}
```

- `message` — структура данных
    
- Поля имеют **тип и уникальный номер** (`id = 1`)
    
- Номера нужны для бинарной сериализации и версионности
    

---

## 🔷 Почему номера полей важны

- Старые клиенты могут игнорировать новые поля
    
- Можно добавлять новые поля без нарушения совместимости
    
- Порядок полей в бинарной записи **не имеет значения**
    

---

## 🔷 Преимущества protobuf

|Преимущество|Пояснение|
|---|---|
|Компактность|Бинарный формат меньше JSON/XML|
|Скорость|Быстрая сериализация/десериализация|
|Типизация|Строго типизированные поля|
|Версионность|Добавление новых полей без поломки старых клиентов|
|Кроссплатформенность|Генерация кода на Java, C++, Python, Go, JS и др.|

---

## 🔷 Пример сериализации/десериализации

**Python пример:**

```python
import user_pb2

# Создание
user = user_pb2.User(id=10, name="John", email="john@example.com")

# Сериализация в бинарный формат
data = user.SerializeToString()

# Десериализация
user2 = user_pb2.User()
user2.ParseFromString(data)
print(user2.name)  # John
```

---

## 🔷 Где используется

- **gRPC** — protobuf стандартно используется как транспортный формат
    
- Микросервисы
    
- Внутренние обмены данными между системами
    
- Любые приложения, где важна **скорость и компактность**
    

---

## 🔷 Protobuf vs JSON vs XML

||JSON|XML|Protobuf|
|---|---|---|---|
|Формат|Текстовый|Текстовый|Бинарный|
|Размер|Большой|Очень большой|Маленький|
|Скорость|Средняя|Медленная|Высокая|
|Типизация|Нет|Нет|Строгая|
|Версионность|Нет|Нет|Да|
|Читаемость|Читаемый|Читаемый|Нет (бинарный)|
