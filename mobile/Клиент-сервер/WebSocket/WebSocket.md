## 🔷 Что такое WebSocket

**WebSocket** — это протокол, который обеспечивает **постоянное двустороннее соединение** между клиентом и сервером поверх TCP.

> В отличие от HTTP, соединение **не закрывается после запроса**.

Клиент и сервер могут **в любой момент** отправлять данные друг другу.

---

## 🔷 Проблема HTTP, которую решает WebSocket

HTTP работает так:

```
запрос → ответ → соединение закрыто
```

Если серверу нужно что-то «сообщить» клиенту:

- polling (постоянные запросы)
    
- long polling
    
- костыли
    

WebSocket решает это **одним постоянным каналом**.

---

## 🔷 Как устанавливается соединение

Сначала обычный HTTP-запрос с «апгрейдом»:

```http
GET /chat HTTP/1.1
Host: example.com
Upgrade: websocket
Connection: Upgrade
Sec-WebSocket-Key: x3JJHMbDL1EzLkh9GBhXDw==
Sec-WebSocket-Version: 13
```

Сервер отвечает:

```http
HTTP/1.1 101 Switching Protocols
Upgrade: websocket
Connection: Upgrade
```

После этого начинается **уже не HTTP**, а протокол WebSocket.

---

## 🔷 Что происходит дальше

Соединение остаётся открытым:

```
Клиент ⇄ Сервер ⇄ Клиент ⇄ Сервер
```

Без новых HTTP-запросов.

---

## 🔷 Где используется WebSocket

- Чаты
    
- Онлайн-игры
    
- Биржи, графики в реальном времени
    
- Уведомления
    
- GraphQL Subscriptions
    
- Коллаборативные редакторы (Google Docs)
    

---

## 🔷 Особенности WebSocket

|Особенность|Пояснение|
|---|---|
|Постоянное соединение|Нет повторных запросов|
|Двусторонняя связь|Сервер сам шлёт данные|
|Низкая задержка|Нет HTTP overhead|
|Работает поверх TCP|Надёжная доставка|

---

## 🔷 WebSocket vs HTTP

|HTTP|WebSocket|
|---|---|
|Запрос-ответ|Постоянный канал|
|Клиент инициирует|Обе стороны|
|Закрывается после ответа|Держится открытым|
|Подходит для REST|Подходит для real-time|
