Here is a comprehensive, practical guide to **Unit**, **Integration**, and **E2E** testing in the modern Java ecosystem, complete with tooling recommendations and code examples.

---

### 1. The Testing Pyramid (The Big Picture)
- **Unit Tests** (Base): Many, fast, cheap. Test single classes in isolation.
- **Integration Tests** (Middle): Fewer, slower. Test how modules talk to databases, message queues, or other services.
- **E2E Tests** (Top): Fewest, slowest, expensive. Test the entire running application from the outside (UI or API).

---

### 2. Unit Tests (JUnit 5 + Mockito)
**Goal:** Verify business logic *without* touching the file system, network, or database.
**Rules:** If it hits a database or an API, **mock it**.

**Tools:** `JUnit Jupiter`, `Mockito`, `AssertJ`.

**Example:** Testing a `UserService` while mocking the `UserRepository`.

```java
import org.junit.jupiter.api.Test;
import org.junit.jupiter.api.extension.ExtendWith;
import org.mockito.InjectMocks;
import org.mockito.Mock;
import org.mockito.junit.jupiter.MockitoExtension;

import java.util.Optional;

import static org.junit.jupiter.api.Assertions.*;
import static org.mockito.Mockito.*;

@ExtendWith(MockitoExtension.class) // Activates Mockito
class UserServiceTest {

    @Mock
    private UserRepository userRepository; // The fake dependency

    @InjectMocks
    private UserService userService; // The real service, injected with mocks

    @Test
    void getUserById_ShouldReturnUser_WhenUserExists() {
        // Arrange (Given)
        Long userId = 1L;
        User mockUser = new User(userId, "John Doe");
        when(userRepository.findById(userId)).thenReturn(Optional.of(mockUser));

        // Act (When)
        User result = userService.getUserById(userId);

        // Assert (Then)
        assertNotNull(result);
        assertEquals("John Doe", result.getName());
        verify(userRepository, times(1)).findById(userId);
    }

    @Test
    void getUserById_ShouldThrowException_WhenUserNotFound() {
        Long userId = 99L;
        when(userRepository.findById(userId)).thenReturn(Optional.empty());

        assertThrows(UserNotFoundException.class, () -> userService.getUserById(userId));
    }
}
```

### 3. Integration Tests (Spring Boot + Testcontainers)
**Goal:** Verify that your code interacts correctly with external infrastructure (Databases, Redis, Kafka, or third-party REST APIs).

**Crucial Shift:** Do **not** use H2 (in-memory DB) for integration tests if you use PostgreSQL/MySQL in production. H2 has different SQL dialects and locking mechanisms. Use **Testcontainers** to spin up a real database in a Docker container.

**Tools:** `@SpringBootTest` (slices), `Testcontainers`, `WireMock` (for external APIs), `@DataJpaTest`.

#### A) Database Integration (Repository Layer)
Testing the actual SQL and Hibernate mapping against a real PostgreSQL database.

```java
import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.autoconfigure.jdbc.AutoConfigureTestDatabase;
import org.springframework.boot.test.autoconfigure.orm.jpa.DataJpaTest;
import org.springframework.test.context.DynamicPropertyRegistry;
import org.springframework.test.context.DynamicPropertySource;
import org.testcontainers.containers.PostgreSQLContainer;
import org.testcontainers.junit.jupiter.Container;
import org.testcontainers.junit.jupiter.Testcontainers;

import static org.assertj.core.api.Assertions.assertThat;

@DataJpaTest // Loads only JPA components, not the whole web layer
@Testcontainers
@AutoConfigureTestDatabase(replace = AutoConfigureTestDatabase.Replace.NONE) // Disable H2
class UserRepositoryIntegrationTest {

    @Container
    static PostgreSQLContainer<?> postgres = new PostgreSQLContainer<>("postgres:15")
            .withDatabaseName("testdb")
            .withUsername("test")
            .withPassword("test");

    @DynamicPropertySource
    static void properties(DynamicPropertyRegistry registry) {
        registry.add("spring.datasource.url", postgres::getJdbcUrl);
        registry.add("spring.datasource.username", postgres::getUsername);
        registry.add("spring.datasource.password", postgres::getPassword);
    }

    @Autowired
    private UserRepository userRepository;

    @Test
    void shouldSaveAndFindUser() {
        User user = new User(null, "integration@test.com");
        User saved = userRepository.save(user);

        var found = userRepository.findById(saved.getId());
        assertThat(found).isPresent();
        assertThat(found.get().getEmail()).isEqualTo("integration@test.com");
    }
}
```

#### B) Service/Controller Integration (Slice Tests)
Testing the Web layer (`@WebMvcTest`) or broader context (`@SpringBootTest`) with mocked collaborators.

```java
@WebMvcTest(UserController.class) // Only loads web layer, not the entire context
class UserControllerIntegrationTest {

    @Autowired
    private MockMvc mockMvc;

    @MockBean // Mock the service that the controller calls
    private UserService userService;

    @Test
    void getEndpoint_ShouldReturn200() throws Exception {
        when(userService.getUserById(1L)).thenReturn(new User(1L, "Mock Name"));

        mockMvc.perform(get("/api/users/1"))
                .andExpect(status().isOk())
                .andExpect(jsonPath("$.name").value("Mock Name"));
    }
}
```

### 4. End-to-End (E2E) Tests (RestAssured + Testcontainers)
**Goal:** Test the *entire* system as a black box. Spin up the full Spring Boot application (with a real test database) and send real HTTP requests to it. No mocks (except for external 3rd-party payments/APIs which you might stub).

**Tools:** `RestAssured` (for REST APIs), `Playwright` or `Selenium` (for UI), `Cucumber` (BDD).

**Example:** Firing up the full app on a random port and testing the full CRUD lifecycle.

```java
import io.restassured.RestAssured;
import io.restassured.http.ContentType;
import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.Test;
import org.springframework.boot.test.context.SpringBootTest;
import org.springframework.boot.test.web.server.LocalServerPort;
import org.springframework.test.context.DynamicPropertyRegistry;
import org.springframework.test.context.DynamicPropertySource;
import org.testcontainers.containers.PostgreSQLContainer;
import org.testcontainers.junit.jupiter.Container;
import org.testcontainers.junit.jupiter.Testcontainers;

import static io.restassured.RestAssured.given;
import static org.hamcrest.Matchers.equalTo;
import static org.hamcrest.Matchers.notNullValue;

@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT) // Full context
@Testcontainers
class UserE2ETest {

    @Container
    static PostgreSQLContainer<?> postgres = new PostgreSQLContainer<>("postgres:15")
            .withDatabaseName("e2e_db")
            .withUsername("e2e")
            .withPassword("e2e");

    @DynamicPropertySource
    static void properties(DynamicPropertyRegistry registry) {
        registry.add("spring.datasource.url", postgres::getJdbcUrl);
        // Ensure Liquibase/Hibernate runs on startup
    }

    @LocalServerPort
    private Integer port;

    @BeforeEach
    void setUp() {
        RestAssured.port = port;
        RestAssured.baseURI = "http://localhost";
    }

    @Test
    void fullCreateAndGetFlow_ShouldSucceed() {
        // 1. Create User (POST)
        String userId = given()
                .contentType(ContentType.JSON)
                .body("""
                        {
                            "name": "E2E Tester",
                            "email": "e2e@test.com"
                        }
                        """)
                .when()
                .post("/api/users")
                .then()
                .statusCode(201)
                .body("id", notNullValue())
                .extract()
                .path("id")
                .toString();

        // 2. Retrieve User (GET)
        given()
                .when()
                .get("/api/users/" + userId)
                .then()
                .statusCode(200)
                .body("name", equalTo("E2E Tester"))
                .body("email", equalTo("e2e@test.com"));
    }
}
```

### 5. How to organize them in Maven / Gradle
To keep builds fast, separate them:

- **Unit Tests:** Run automatically on `mvn test` (convention: `*Test.java`).
- **Integration Tests:** Run on `mvn verify` (convention: `*IT.java`). Use the `maven-failsafe-plugin`.
- **E2E Tests:** Often run in a separate CI job or manually against a staging environment, as they require heavy resources.

### 6. Summary Comparison Table

| Feature | Unit Test | Integration Test | E2E Test |
| :--- | :--- | :--- | :--- |
| **Scope** | Single class/method | Layer communication (DB, Network) | Entire application stack |
| **Mocks** | **Yes** (everything external) | Minimal (only external 3rd-party APIs) | **No** (test everything real) |
| **Speed** | Milliseconds | Seconds | Tens of seconds/minutes |
| **Environment** | Local JVM | Local JVM + Docker (Testcontainers) | Staging / Dedicated CI environment |
| **Failure** | Pinpoints exact bug instantly | Finds data mapping/config errors | Finds deployment/config/hardware issues |
| **Rule of thumb** | Write as many as possible | Write for critical paths | Write only for "happy paths" and critical business flows |

---

### 7. Pro-Tips (Best Practices)
1.  **Testcontainers over H2:** Stop using H2 for integration tests. It lies to you about SQL syntax and transaction isolation. Use Testcontainers.
2.  **External APIs:** If your app calls Stripe/Twilio, use **WireMock** inside your Integration tests, not Mockito. This validates that your JSON serialization matches the actual API contract.
3.  **`@SpringBootTest` Scoping:** Don't use `@SpringBootTest` for everything. It loads the *entire* context. Use `@DataJpaTest`, `@WebMvcTest`, or `@JsonTest` whenever possible to keep integration tests fast.
4.  **Clean State:** For E2E and Integration tests, always reset the database schema between test classes (e.g., using `@DirtiesContext` or transactional rollbacks with `@Transactional` on tests).
