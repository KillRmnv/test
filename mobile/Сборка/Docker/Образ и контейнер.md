## Образ — это **шаблон**, а не сам код

Представь, что образ — это **форма для выпечки печенья**:
- Форма (образ) содержит ВСЁ необходимое: форму звездочки, рельеф, размер
- Но это не само печенье!
- Из одной формы можно испечь тысячи одинаковых печенек (контейнеров)

**Образ содержит:**
- Файловую систему (код, библиотеки, конфиги)
- Переменные окружения (по умолчанию)
- Команду для запуска (CMD)
- Метаданные (автор, порты, volumes)

Но! Образ **не запущен** — это просто "слепок" файловой системы и настроек.

## А что же тогда **контейнер**?

Контейнер — это **запущенный экземпляр образа**:
- Берем образ (форму для печенья)
- Добавляем "тесто" (процессы, память, сеть)
- Получаем печенье (работающее приложение)

Контейнер = Образ + Тонкий слой записи + Запущенные процессы

## Давай посмотрим вживую

### 1. Создадим свой образ
```dockerfile
FROM alpine:latest
RUN echo "#!/bin/sh" > /app.sh && \
    echo 'echo "Привет, я образ!"' >> /app.sh && \
    chmod +x /app.sh
CMD ["/app.sh"]
```

Собираем:
```bash
docker build -t myapp .
```

### 2. Смотрим на образ
```bash
# Образ существует, но ничего не делает
$ docker image ls
REPOSITORY   TAG       IMAGE ID       CREATED         SIZE
myapp        latest    abc123def456   1 minute ago    7.05MB

# Можно посмотреть, что внутри образа
$ docker run -it --rm myapp cat /app.sh
#!/bin/sh
echo "Привет, я образ!"
```

### 3. Создаем контейнеры из образа
```bash
# Запускаем контейнер №1
$ docker run --name cont1 myapp
Привет, я образ!

# Запускаем контейнер №2
$ docker run --name cont2 myapp
Привет, я образ!

# Смотрим контейнеры
$ docker ps -a
CONTAINER ID   IMAGE     COMMAND    CREATED          STATUS
def456...      myapp     "/app.sh"  10 seconds ago   Exited (0)  # cont2
ghi789...      myapp     "/app.sh"  20 seconds ago   Exited (0)  # cont1
```

**Важно:** Один образ — много контейнеров. Все контейнеры одинаковые в момент старта, но могут становиться разными.

## Изменяемость vs Неизменяемость

### Образ — **неизменяемый** (read-only)
```bash
# Нельзя изменить образ
$ docker run -it --rm myapp rm /app.sh
rm: remove /app.sh? y
# Файл удален В КОНТЕЙНЕРЕ, но не в образе!

# Запускаем новый контейнер
$ docker run -it --rm myapp ls -l /app.sh
-rwxr-xr-x    1 root     root            38 /app.sh   # Файл на месте!
```

### Контейнер — **изменяемый** (добавляет слой записи)
Когда контейнер запускается, Docker:
1. Берет образ (read-only слои)
2. Добавляет сверху тонкий слой для записи
3. Все изменения пишутся в этот слой, оригинальный образ НЕ ТРОГАЕТСЯ

```bash
# Запускаем контейнер
$ docker run -it myapp sh

# Внутри контейнера создаем файл
/app # echo "новые данные" > /новый-файл.txt

# Выходим
/app # exit

# Контейнер остановлен, но его слой с изменениями сохранен
$ docker start -a контейнер
# ... файл /новый-файл.txt все еще там!
```

## Что хранится в образе?

### 1. **Слои файловой системы**
```dockerfile
FROM ubuntu:22.04           # Слой 1: базовая ОС
RUN apt-get update          # Слой 2: обновленные пакеты
COPY app.py /app/           # Слой 3: исходный код
RUN pip install flask       # Слой 4: зависимости Python
CMD ["python", "app.py"]     # Метаданные, не слой
```

### 2. **Метаданные**
```bash
$ docker inspect myapp:latest
[
    {
        "Config": {
            "Env": ["PATH=/usr/local/bin:..."],  # Переменные окружения
            "Cmd": ["python", "app.py"],          # Команда по умолчанию
            "WorkingDir": "/app",                  # Рабочая директория
            "ExposedPorts": {"5000/tcp": {}}       # Какие порты открыты
        }
    }
]
```

### 3. **НЕ хранится в образе**
- Запущенные процессы (это в контейнере)
- Данные, которые меняются (пишутся в слой контейнера)
- Состояние сети (IP-адреса назначаются контейнерам)

## Аналогия для запоминания

| Концепция | Аналогия | Особенности |
|-----------|----------|-------------|
| **Dockerfile** | Рецепт печенья | Инструкция, как готовить |
| **Образ (Image)** | Форма для печенья | Неизменяемая, многоразовая |
| **Контейнер (Container)** | Готовое печенье | Можно есть (запускать), можно изменить (посыпать сахарной пудрой) |
| **Registry (Docker Hub)** | Магазин форм | Где хранятся и продаются формы |

## Почему это гениально?

### 1. Экономия места
```bash
# 10 контейнеров на базе одного образа:
Образ Ubuntu: 500 МБ
10 контейнеров: по 5 МБ на слой записи = 50 МБ
ИТОГО: 550 МБ вместо 5 ГБ!
```

### 2. Скорость
```bash
# Запуск контейнера = секунды, потому что:
# - Образ уже скачан
# - Нужно только создать тонкий слой записи
# - Файлы не копируются, используются через copy-on-write
```

### 3. Надежность
Образ неизменяем = "Золотой стандарт". Если он работает сегодня, будет работать и завтра. Никто случайно не изменит файлы в образе.
