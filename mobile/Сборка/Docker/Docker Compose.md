## Что такое Docker Compose?

**Docker Compose** — это инструмент для определения и запуска многоконтейнерных приложений Docker . Вместо того чтобы писать длинные команды `docker run ...` для каждого контейнера, вы описываете всё приложение (веб-сервер, базу данных, кэш, очередь задач) в одном файле и запускаете одной командой .

### Основная идея
*   **Без Compose**: Вы пишете несколько команд `docker build` и `docker run`, вручную создаете сеть для связи контейнеров, подключаете тома. Это работает, но становится сложно при росте приложения.
*   **С Compose**: Вы создаете файл конфигурации (обычно `compose.yaml`), где описываете все сервисы, их настройки, сети и тома. Затем командой `docker compose up` запускаете всё сразу .

## Ключевая концепция: Проект и Сервисы

Docker Compose вводит два важных понятия:

1.  **Сервис (Service)** — это определение контейнера. Например, "база данных PostgreSQL" или "веб-приложение на Python". В файле Compose вы описываете, из какого образа его создать, какие порты открыть, какие переменные окружения задать.
2.  **Проект (Project)** — это группа сервисов, которые вместе образуют приложение. Compose автоматически изолирует проекты друг от друга, создавая для каждого отдельные сети и добавляя префикс с именем проекта к именам контейнеров .

## Структура compose.yaml

Давайте разберем типичный файл Compose на примере приложения "Flask + Redis", который приводится в официальной документации :

```yaml
# Это наш файл compose.yaml
services:
  web:                          # Название первого сервиса
    build: .                    # Собрать образ из Dockerfile в текущей папке
    ports:
      - "8000:5000"             # Проброс порта: порт_хоста:порт_контейнера
    develop:                    # Опции для разработки (watch mode)
      watch:
        - action: sync
          path: .
          target: /code

  redis:                        # Название второго сервиса
    image: "redis:alpine"        # Использовать готовый образ из Docker Hub
```

### Ключевые элементы

*   **`services`**: Родительский раздел, где перечисляются все сервисы приложения.
*   **`build`**: Указывает путь к папке с `Dockerfile`. Compose соберет образ перед запуском .
*   **`image`**: Указывает готовый образ для скачивания с Docker Hub или другого registry .
*   **`ports`**: Пробрасывает порты с хоста в контейнер (формат `"HOST:CONTAINER"`) .
*   **`environment`**: Задает переменные окружения внутри контейнера .
*   **`volumes`**: Монтирует папки с хоста или именованные тома в контейнер .
*   **`depends_on`**: Указывает порядок запуска. Например, `flask` сервис запустится только после `redis` .

## Основные команды Docker Compose

Вот команды, которые вы будете использовать чаще всего. Обратите внимание, что современная версия (V2) вызывается как `docker compose` (через пробел), а не `docker-compose` (с дефисом) .

| Команда | Описание | Пример |
|---------|----------|---------|
| **`up`** | Создает и запускает все контейнеры, описанные в файле. Флаг `-d` означает "detached" (в фоне). | `docker compose up -d`  |
| **`down`** | Останавливает и удаляет все контейнеры, сети, созданные командой `up`. Флаг `-v` удалит еще и тома. | `docker compose down -v`  |
| **`ps`** | Показывает список запущенных контейнеров текущего проекта. | `docker compose ps`  |
| **`logs`** | Показывает логи всех сервисов или конкретного. | `docker compose logs web`  |
| **`exec`** | Выполняет команду внутри уже запущенного контейнера. | `docker compose exec web bash`  |
| **`build`** | Пересобирает образы для сервисов, где указана директива `build`. | `docker compose build`  |
| **`stop`** | Останавливает контейнеры, но не удаляет их. | `docker compose stop`  |
| **`start`** | Запускает ранее остановленные контейнеры. | `docker compose start`  |

### Работа с несколькими файлами
Docker Compose позволяет использовать несколько файлов конфигурации. Это удобно для разделения настроек (например, общие настройки в `compose.yaml`, настройки для разработки в `compose.override.yaml`) . Можно указать несколько файлов через флаг `-f`:
```bash
docker compose -f compose.yaml -f compose.prod.yaml up -d
```
В этом случае второй файл дополняет или переопределяет настройки первого .

## Зачем это нужно?

### 1. Локальная разработка (Development)
Это главный сценарий использования. Compose позволяет поднять всё окружение проекта одной командой. Новый разработчик больше не читает 10-страничную инструкцию "Как запустить проект", а просто пишет `docker compose up` .

### 2. Автоматическое тестирование (CI)
В пайплайнах непрерывной интеграции Compose используется для быстрого создания изолированного окружения, прогона тестов и его последующего удаления .
```bash
docker compose up -d
./run_tests.sh
docker compose down
```

### 3. Демонстрация (Demo / Staging)
Мгновенно поднять демо-версию продукта со всеми зависимостями.
